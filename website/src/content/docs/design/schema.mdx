Schema defines the structure of interaction data. Before storing data, define how edges are structured, what properties they have, and how they can be queried.

See [Core Concepts](/design/concepts/) for background.

## Schema Hierarchy

```
Service
├── Label (edge schema)
│   ├── Schema (source, target, properties)
│   ├── Indexes (for querying)
│   └── Storage (physical storage)
└── Alias (alternative name for Label)
```

- **Service** groups related Labels and Aliases
- **Label** defines the schema for edges
- **Alias** provides an alternative name for a Label
- **Label** references a Storage for physical data

## Service

A namespace that contains labels and aliases.

| Property | Description                            |
| -------- | -------------------------------------- |
| name     | Service identifier (e.g., `myservice`) |
| desc     | Description                            |
| active   | Whether active                         |

Example: an e-commerce service might contain labels for `likes`, `recent_views`, and `purchases`.

## Label

Defines the schema for edges—source, target, properties, and indexes.

| Property | Description                       |
| -------- | --------------------------------- |
| name     | Format: `service.label`           |
| schema   | Edge structure (src, tgt, fields) |
| indices  | Indexes for querying              |
| storage  | Storage name                      |
| dirType  | Direction type                    |
| active   | Whether active                    |

### Schema Definition

- **src**: source type (STRING, LONG) — who
- **tgt**: target type (STRING, LONG) — what
- **fields**: properties, each with name, type, nullable

**Example: Recent Views**

```
source: user_id (LONG)
target: product_id (LONG)
properties:
  - created_at (LONG)
```

**Example: Reactions**

```
source: user_id (LONG)
target: product_id (LONG)
properties:
  - created_at (LONG)
  - reaction_type (STRING)
```

### Index Definition

Indexes enable efficient querying. Each index has a name and a list of fields with sort order.

```
indexes:
  - name: by_created_at
    fields: [created_at DESC]
  - name: by_type_and_time
    fields: [reaction_type ASC, created_at DESC]
```

Indexes are pre-computed at write time. See [Query](/design/query/).

## Alias

An alternative name for a label.

| Property | Description             |
| -------- | ----------------------- |
| name     | Format: `service.alias` |
| target   | The label it points to  |
| active   | Whether active          |

Useful for gradual migrations or domain-specific naming.

## Storage

Where data is physically stored.

| Property | Description                  |
| -------- | ---------------------------- |
| type     | Storage type (e.g., `hbase`) |
| name     | Storage identifier           |
| conf     | Configuration map            |
| active   | Whether active               |

See [Datastore](/design/datastore/).

## Naming Conventions

| Type    | Format            | Example             |
| ------- | ----------------- | ------------------- |
| Service | Simple identifier | `myservice`         |
| Label   | `service.label`   | `myservice.likes`   |
| Alias   | `service.alias`   | `myservice.friends` |
| Storage | Simple identifier | `hbase-storage-1`   |

## API Versions

Schema uses v2 API. Mutation and query use v3.

| v2 Term | v3 Term  |
| ------- | -------- |
| Service | Database |
| Label   | Table    |

Schema v3 is planned.
