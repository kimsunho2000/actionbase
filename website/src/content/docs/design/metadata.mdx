Metadata in Actionbase defines the structure and configuration of your graph database, including databases, tables, and storage settings.

## Concepts

### Database

A database is a top-level container that groups related tables together. It provides isolation and organization for your data.

### Table

Tables define the structure of your graph data. Actionbase supports different table types:

- **EdgeTable**: Stores relationships between entities
- **VertexTable**: Stores entity data
- **MultiEdgeTable**: Stores multiple relationships between the same entities

### Storage

Storage configuration defines how data is physically stored in the underlying storage backend (e.g., HBase). This includes settings for replication, compression, and region management.

## Metadata Operations

### Creating a Database

```bash
actionbase database create --name mydb
```

### Creating a Table

```bash
actionbase table create \
  --database mydb \
  --name mytable \
  --type EdgeTable
```

### Listing Metadata

```bash
# List all databases
actionbase database list

# List tables in a database
actionbase table list --database mydb
```

## Schema Evolution

Actionbase supports flexible schema evolution. You can add new properties to vertices and edges without requiring schema migrations, making it easy to adapt to changing requirements.
