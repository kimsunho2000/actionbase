서비스, 레이블, 별칭, 쿼리를 관리합니다.

## 1. 서비스

서비스는 레이블과 별칭을 포함하는 논리적 그룹으로, 관련 그래프 구조에 대한 조직화와 격리를 제공합니다.

### 1.1 서비스 조회

이름으로 단일 서비스를 조회합니다.

#### 엔드포인트

```
GET /graph/v2/service/{service}
```

#### 파라미터

| 위치   | 파라미터      | 필수 여부 | 설명                              |
| ------ | ------------- | --------- | --------------------------------- |
| Header | Authorization | 선택      | 인증 키 (향후 사용을 위해 예약됨) |
| Path   | service       | 필수      | 서비스 이름                       |

#### 응답 타입

[ServiceEntity](#serviceentity) - 서비스 정보

#### 요청 예시

| 파라미터      | 값           |
| ------------- | ------------ |
| Authorization | YOUR_API_KEY |
| service       | myservice    |

```bash
# GET /graph/v2/service/myservice
curl -X GET \
    "http://ab.example.com/graph/v2/service/myservice" \
    -H "Authorization: YOUR_API_KEY"
```

#### 응답 예시

```json
{
  "active": true,
  "name": "myservice",
  "desc": "My service description"
}
```

### 1.2 전체 서비스 조회

모든 서비스를 조회합니다.

#### 엔드포인트

```
GET /graph/v2/service
```

#### 파라미터

| 위치   | 파라미터      | 필수 여부 | 설명                              |
| ------ | ------------- | --------- | --------------------------------- |
| Header | Authorization | 선택      | 인증 키 (향후 사용을 위해 예약됨) |

#### 응답 타입

[DdlPage](#ddlpage) of [ServiceEntity](#serviceentity) - 페이지네이션된 서비스 목록

#### 요청 예시

```bash
# GET /graph/v2/service
curl -X GET \
    "http://ab.example.com/graph/v2/service" \
    -H "Authorization: YOUR_API_KEY"
```

#### 응답 예시

```json
{
  "count": 2,
  "content": [
    {
      "active": true,
      "name": "myservice",
      "desc": "My service description"
    },
    {
      "active": true,
      "name": "anotherservice",
      "desc": "Another service"
    }
  ]

```

### 1.3 서비스 생성

새 서비스를 생성합니다.

#### 엔드포인트

```

```

#### 파라미터

| 위치   | 파라미터      | 필수 여부 | 설명                              |
| ------ | ------------- | --------- | --------------------------------- |
| Header | Authorization | 선택      | 인증 키 (향후 사용을 위해 예약됨) |
| Path   | service       | 필수      | 서비스 이름                       |
| Body   | desc          | 필수      | 서비스 설명                       |

#### 요청 바디

[ServiceCreateRequest](#servicecreaterequest) - 서비스 생성 페이로드

#### 응답 타입

[DdlStatus](#ddlstatus) of [ServiceEntity](#serviceentity) - 서비스 생성 상태

```bash
# POST /graph/v2/service/myservice
curl -X POST \
    "http://ab.example.com/graph/v2/service/myservice" \
    -H "Authorization: YOUR_API_KEY" \
    -H "Content-Type: application/json" \
    -d '{
      "desc": "My service description"
    }'
```

#### 응답 예시

```json
{
  "status": "CREATED",
  "result": {
    "active": true,
    "name": "myservice",

  }
```


## 2. 레이블

레이블은 그래프 엣지의 스키마를 정의하며, 소스 및 타겟 노드 타입, 엣지 프로퍼티, 인덱스를 포함한 관계의 구조를 지정합니다.

### 2.1 레이블 조회

이름으로 단일 레이블을 조회합니다.

#### 엔드포인트

```
GET /graph/v2/service/{service}/label/{label}
```

#### 파라미터

| 위치   | 파라미터      | 필수 여부 | 설명                              |
| ------ | ------------- | --------- | --------------------------------- |
| Header | Authorization | 선택      | 인증 키 (향후 사용을 위해 예약됨) |
| Path   | service       | 필수      | 서비스 이름                       |
| Path   | label         | 필수      | 레이블 이름                       |

#### 응답 타입

[LabelEntity](#labelentity) - 레이블 정보

#### 요청 예시

```bash
# GET /graph/v2/service/myservice/label/follows
curl -X GET \
    "http://ab.example.com/graph/v2/service/myservice/label/follows" \
    -H "Authorization: YOUR_API_KEY"
```

#### 응답 예시

```json
{
  "active": true,
  "name": "myservice.follows",
  "desc": "User follows relationship",
  "type": "HASH",
  "schema": {
    "src": {
      "type": "LONG",
      "desc": "User ID"
    },
    "tgt": {
      "type": "LONG",
      "desc": "User ID"
    },
    "fields": [
      {
        "name": "created_at",
        "type": "LONG",
        "nullable": false,
        "desc": "Creation timestamp"
      }
    ]
  },
  "dirType": "OUT",
  "storage": "hbase-storage-1",
  "indices": [],
  "groups": [],
  "event": false,
  "readOnly": false,
  "mode": "SYNC"
}
```

### 2.3 레이블 생성

새 레이블을 생성합니다.

#### 엔드포인트

```
POST /graph/v2/service/{service}/label/{label}
```

#### 파라미터

| 위치   | 파라미터      | 필수 여부 | 설명                                    |
| ------ | ------------- | --------- | --------------------------------------- |
| Header | Authorization | 선택      | 인증 키 (향후 사용을 위해 예약됨)       |
| Path   | service       | 필수      | 서비스 이름                             |
| Path   | label         | 필수      | 레이블 이름                             |
| Body   | desc          | 필수      | 레이블 설명                             |
| Body   | type          | 필수      | 레이블 타입 (HASH, INDEXED, MULTI_EDGE) |
| Body   | schema        | 필수      | 엣지 스키마 정의                        |
| Body   | dirType       | 필수      | 방향 타입 (OUT, IN, BOTH)               |
| Body   | storage       | 필수      | 스토리지 이름                           |
| Body   | groups        | 선택      | 그룹 목록                               |
| Body   | indices       | 선택      | 인덱스 목록                             |
| Body   | event         | 선택      | 이벤트 활성화 여부 (기본값: false)      |
| Body   | readOnly      | 선택      | 레이블 읽기 전용 여부 (기본값: false)   |
| Body   | mode          | 선택      | 뮤테이션 모드 (기본값: SYNC)            |

#### 요청 바디

[LabelCreateRequest](#labelcreaterequest) - 레이블 생성 페이로드

#### 응답 타입

[DdlStatus](#ddlstatus) of [LabelEntity](#labelentity) - 레이블 생성 상태

#### 요청 예시

```bash
# POST /graph/v2/service/myservice/label/follows
curl -X POST \
    "http://ab.example.com/graph/v2/service/myservice/label/follows" \
    -H "Authorization: YOUR_API_KEY" \
    -H "Content-Type: application/json" \
    -d '{
      "desc": "User follows relationship",
      "type": "HASH",
      "schema": {
        "src": {
          "type": "LONG",
          "desc": "User ID"
        },
        "tgt": {
          "type": "LONG",
          "desc": "User ID"
        },
        "fields": [
          {
            "name": "created_at",
            "type": "LONG",
            "nullable": false,
            "desc": "Creation timestamp"
          }
        ]
      },
      "dirType": "OUT",
      "storage": "hbase-storage-1",
      "indices": [],
      "groups": [],
      "event": false,
      "readOnly": false,
      "mode": "SYNC"
    }'
```


#### 응답 예시

