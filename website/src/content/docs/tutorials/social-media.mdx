This tutorial demonstrates how to build a social media application using Actionbase, including features like user relationships, activity feeds, and engagement metrics.

## Overview

We'll build a social media application with the following features:

- User profiles
- Follow/unfollow relationships
- Activity feeds
- Like and comment tracking
- Engagement metrics

## Data Model

### Users (Vertices)

```json
{
  "id": "user123",
  "properties": {
    "username": "johndoe",
    "email": "john@example.com",
    "createdAt": "2025-01-01T00:00:00Z"
  }
}
```

### Follows (Edges)

```json
{
  "source": "user123",
  "target": "user456",
  "properties": {
    "createdAt": "2025-01-01T00:00:00Z"
  }
}
```

### Likes (Edges)

```json
{
  "source": "user123",
  "target": "post789",
  "properties": {
    "createdAt": "2025-01-01T00:00:00Z"
  }
}
```

## Implementation

### Step 1: Create Database and Tables

```bash
# Create database
actionbase database create --name socialmedia

# Create users table (VertexTable)
actionbase table create \
  --database socialmedia \
  --name users \
  --type VertexTable

# Create follows table (EdgeTable)
actionbase table create \
  --database socialmedia \
  --name follows \
  --type EdgeTable

# Create likes table (EdgeTable)
actionbase table create \
  --database socialmedia \
  --name likes \
  --type EdgeTable
```

### Step 2: Create Users

```json
POST /graph/v3/vertex/mutate
{
  "database": "socialmedia",
  "table": "users",
  "id": "user123",
  "properties": {
    "username": "johndoe",
    "email": "john@example.com",
    "createdAt": "2025-01-01T00:00:00Z"
  }
}
```

### Step 3: Create Follow Relationships

```json
POST /graph/v3/edge/mutate
{
  "database": "socialmedia",
  "table": "follows",
  "source": "user123",
  "target": "user456",
  "properties": {
    "createdAt": "2025-01-01T00:00:00Z"
  }
}
```

### Step 4: Query Followers

Get all users that follow a specific user:

```json
POST /graph/v3/edge/scan
{
  "database": "socialmedia",
  "table": "follows",
  "target": "user123",
  "direction": "incoming"
}
```

### Step 5: Count Followers

Get follower count (uses write-time optimization):

```json
GET /graph/v3/edge/count?database=socialmedia&table=follows&target=user123
```

### Step 6: Activity Feed

Get activity feed for a user (posts from users they follow):

```json
POST /graph/v3/edge/scan
{
  "database": "socialmedia",
  "table": "follows",
  "source": "user123",
  "direction": "outgoing",
  "includeTargets": true
}
```

## Performance Benefits

### Write-time Optimization

When users follow each other or like posts, Actionbase automatically:

- Pre-computes follower/following counts
- Maintains indexes for fast lookups
- Updates feed indexes

### Fast Queries

- **Follower count**: &lt;5ms (uses pre-computed counters)
- **Feed generation**: Optimized with indexes
- **Relationship checks**: &lt;1ms lookups

## Advanced Features

### Engagement Metrics

Track engagement using edge properties:

```json
POST /graph/v3/edge/mutate
{
  "database": "socialmedia",
  "table": "likes",
  "source": "user123",
  "target": "post789",
  "properties": {
    "createdAt": "2025-01-01T00:00:00Z",
    "type": "like"
  }
}
```

### Recommendations

Use graph traversal for recommendations:

```json
POST /graph/v3/edge/scan
{
  "database": "socialmedia",
  "table": "follows",
  "source": "user123",
  "depth": 2,
  "direction": "outgoing"
}
```
